2. Pertemuan 1 - Pengenalan Express.js dan REST API
2.1 Capaian Pembelajaran
Mahasiswa mampu menjelaskan fungsi Express.js sebagai backend API server.
Mahasiswa mampu membedakan Next.js API route dengan Express.js sebagai server terpisah.

2.2 Konsep Dasar
Express.js adalah framework web untuk Node.js yang sering digunakan untuk membuat API server. Jika Next.js sering dipakai sebagai fullstack framework yang menggabungkan frontend dan backend dalam satu project, Express.js lebih sering digunakan sebagai backend terpisah yang menyediakan endpoint untuk frontend web, mobile app, atau sistem lain.

3. Pertemuan 2 - Routing, Middleware, dan CRUD Menggunakan Array
3.1 Capaian Pembelajaran
Mahasiswa mampu membuat route dengan method GET, POST, PUT, dan DELETE.
Mahasiswa mampu mengambil parameter dari URL menggunakan req.params.
Mahasiswa mampu membaca body request menggunakan req.body.
Mahasiswa mampu membuat CRUD sederhana menggunakan array sebagai data sementara.
3.2 Mengapa CRUD Array Dulu?
Sebelum menggunakan database, mahasiswa perlu memahami alur CRUD tanpa beban tambahan seperti koneksi database, user database, migrasi, atau error SQL. Array digunakan sebagai simulasi database sementara. Dengan cara ini, mahasiswa fokus pada routing dan logika HTTP terlebih dahulu.
Tujuan pedagogisCRUD array bukan tujuan akhir. Fungsinya sebagai jembatan agar mahasiswa memahami pola Create, Read, Update, Delete sebelum masuk ke query SQL manual.

File app.ts berisi konfigurasi Express, middleware, dan route. File server.ts bertugas menjalankan server. Pemisahan ini membuat struktur project lebih rapi dan mudah dikembangkan.

3.6 Membuat Route CRUD
src/routes/mahasiswa.route.ts
import { Router, Request, Response } from "express";import { mahasiswa, Mahasiswa } from "../data/mahasiswa.data";const router = Router();// READ ALLrouter.get("/", (req: Request, res: Response) => {  res.json({    message: "Data mahasiswa berhasil diambil",    data: mahasiswa,  });});// READ DETAILrouter.get("/:id", (req: Request, res: Response) => {  const id = Number(req.params.id);  const data = mahasiswa.find((item) => item.id === id);  if (!data) {    return res.status(404).json({ message: "Mahasiswa tidak ditemukan" });  }  res.json({    message: "Detail mahasiswa berhasil diambil",    data,  });});// CREATErouter.post("/", (req: Request, res: Response) => {  const { nim, nama, prodi, angkatan } = req.body;  if (!nim || !nama || !prodi || !angkatan) {    return res.status(400).json({      message: "NIM, nama, prodi, dan angkatan wajib diisi",    });  }  const newMahasiswa: Mahasiswa = {    id: mahasiswa.length + 1,    nim,    nama,    prodi,    angkatan: Number(angkatan),  };  mahasiswa.push(newMahasiswa);  res.status(201).json({    message: "Mahasiswa berhasil ditambahkan",    data: newMahasiswa,  });});// UPDATErouter.put("/:id", (req: Request, res: Response) => {  const id = Number(req.params.id);  const { nim, nama, prodi, angkatan } = req.body;  const index = mahasiswa.findIndex((item) => item.id === id);  if (index === -1) {    return res.status(404).json({ message: "Mahasiswa tidak ditemukan" });  }  mahasiswa[index] = {    id,    nim,    nama,    prodi,    angkatan: Number(angkatan),  };  res.json({    message: "Mahasiswa berhasil diperbarui",    data: mahasiswa[index],  });});// DELETErouter.delete("/:id", (req: Request, res: Response) => {  const id = Number(req.params.id);  const index = mahasiswa.findIndex((item) => item.id === id);  if (index === -1) {    return res.status(404).json({ message: "Mahasiswa tidak ditemukan" });  }  const deletedData = mahasiswa.splice(index, 1);  res.json({    message: "Mahasiswa berhasil dihapus",    data: deletedData[0],  });});export default router;

3.8 Middleware Logging Sederhana
Middleware adalah fungsi yang berjalan di antara request dan response. Contoh sederhana adalah mencatat method dan URL setiap request.
app.use((req, res, next) => {  console.log(`${req.method} ${req.url}`);  next();});
Catatan pentingMiddleware harus memanggil next() jika request ingin diteruskan ke route berikutnya. Jika next() tidak dipanggil dan response juga tidak dikirim, request akan menggantung.

4. Pertemuan 3 - Koneksi Express.js ke MySQL dan Query SQL Manual
4.1 Capaian Pembelajaran
Mahasiswa mampu membuat database dan tabel MySQL untuk kebutuhan CRUD.
Mahasiswa mampu menghubungkan Express.js ke MySQL menggunakan mysql2.
Mahasiswa mampu menjalankan query SELECT dan INSERT dari route Express.js.
Mahasiswa memahami penggunaan placeholder (?) untuk mengurangi risiko SQL injection.
4.2 Mengapa Menggunakan mysql2?
Pada tahap dasar, mysql2 digunakan agar mahasiswa berlatih menulis query SQL secara manual dari backend. Driver mysql2 mendukung Promise dan prepared statement sehingga cocok untuk pembelajaran Express.js modern tanpa ORM.
Fokus pertemuan 3Pertemuan ini belum perlu membuat CRUD database lengkap. Fokusnya adalah koneksi database, SELECT, INSERT, dan pemahaman query parameter. CRUD lengkap dengan database dapat dijadikan materi pertemuan berikutnya.
4.3 Membuat Database dan Tabel
Jalankan perintah SQL berikut di MySQL client, phpMyAdmin, DBeaver, atau DataGrip.
CREATE DATABASE db_kampus;USE db_kampus;CREATE TABLE mahasiswa (  id INT AUTO_INCREMENT PRIMARY KEY,  nim VARCHAR(20) NOT NULL UNIQUE,  nama VARCHAR(100) NOT NULL,  prodi VARCHAR(100) NOT NULL,  angkatan INT NOT NULL,  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP);
Tambahkan data awal:
INSERT INTO mahasiswa (nim, nama, prodi, angkatan) VALUES('2201001', 'Ahmad Fauzi', 'Informatika', 2022),('2201002', 'Siti Aminah', 'Sistem Informasi', 2022);

4.6 Membuat Koneksi Database
Buat file src/config/database.ts
import mysql from "mysql2/promise";import dotenv from "dotenv";dotenv.config();const db = mysql.createPool({  host: process.env.DB_HOST,  user: process.env.DB_USER,  password: process.env.DB_PASSWORD,  database: process.env.DB_NAME,  waitForConnections: true,  connectionLimit: 10,});export default db;
Connection pool digunakan agar aplikasi tidak membuat koneksi baru setiap kali ada request. Ini lebih efisien untuk aplikasi web.
4.7 Membuat Route yang Membaca Data dari MySQL
Ubah route mahasiswa agar membaca data dari database. Untuk tahap awal, buat file baru src/routes/mahasiswa-db.route.ts agar mahasiswa dapat membandingkan versi array dan versi database.
import { Router, Request, Response } from "express";import db from "../config/database";const router = Router();router.get("/", async (req: Request, res: Response) => {  try {    const [rows] = await db.query(      "SELECT * FROM mahasiswa ORDER BY id DESC"    );    res.json({      message: "Data mahasiswa berhasil diambil dari database",      data: rows,    });  } catch (error) {    console.error(error);    res.status(500).json({      message: "Terjadi kesalahan server",    });  }});export default router;
Daftarkan route tersebut pada app.ts:
import mahasiswaDbRoutes from "./routes/mahasiswa-db.route";app.use("/api/db/mahasiswa", mahasiswaDbRoutes);
Uji endpoint:
GET http://localhost:3000/api/db/mahasiswa
4.8 Membaca Detail Data dengan Parameter
router.get("/:id", async (req: Request, res: Response) => {  try {    const { id } = req.params;    const [rows]: any = await db.execute(      "SELECT * FROM mahasiswa WHERE id = ?",      [id]    );    if (rows.length === 0) {      return res.status(404).json({        message: "Mahasiswa tidak ditemukan",      });    }    res.json({      message: "Detail mahasiswa berhasil diambil",      data: rows[0],    });  } catch (error) {    console.error(error);    res.status(500).json({      message: "Terjadi kesalahan server",    });  }});
Mengapa pakai tanda tanya (?)Tanda tanya adalah placeholder. Nilai parameter dikirim terpisah dalam array [id]. Pola ini lebih aman dibanding menyambungkan string query secara manual seperti "WHERE id = " + id.
4.9 Menambah Data ke Database
router.post("/", async (req: Request, res: Response) => {  try {    const { nim, nama, prodi, angkatan } = req.body;    if (!nim || !nama || !prodi || !angkatan) {      return res.status(400).json({        message: "NIM, nama, prodi, dan angkatan wajib diisi",      });    }    const [result]: any = await db.execute(      "INSERT INTO mahasiswa (nim, nama, prodi, angkatan) VALUES (?, ?, ?, ?)",      [nim, nama, prodi, angkatan]    );    res.status(201).json({      message: "Mahasiswa berhasil ditambahkan",      data: {        id: result.insertId,        nim,        nama,        prodi,        angkatan,      },    });  } catch (error: any) {    console.error(error);    if (error.code === "ER_DUP_ENTRY") {      return res.status(400).json({        message: "NIM sudah digunakan",      });    }    res.status(500).json({      message: "Terjadi kesalahan server",    });  }});
